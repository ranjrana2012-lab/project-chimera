# Services Status

This page is retained as a compatibility pointer. The previous service-status
table made stale deployment and monitoring claims that are not part of the
current public Phase 1 boundary.

Current public status:

- the Student / Laptop route is the default validation path;
- the operator-console CLI and web route are the public demo surface;
- generated evidence and grant records are private local artifacts;
- advanced services require separate local validation before being advertised.

Use these maintained references:

- `README.md` for the public status statement.
- `docs/guides/STUDENT_LAPTOP_SETUP.md` for setup.
- `docs/demo/service-status.md` for Phase 1 demo status wording.
- `docs/github/WORKFLOW_INVENTORY.md` for GitHub workflow review notes.
