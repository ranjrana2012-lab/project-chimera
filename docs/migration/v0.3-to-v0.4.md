# Migration Guide: v0.3 to v0.4

**Date:** March 6, 2026

## Overview

This guide helps you migrate from Project Chimera v0.3 to v0.4.0. The v0.4.0 release introduces significant architectural improvements including new services, enhanced observability, and improved orchestration capabilities.

## What's New in v0.4.0

### 8 New Services
v0.4.0 adds 8 new microservices running on ports 8000-8007:
- **BSL Agent (Port 8000)**: Behavioral Safety Layer agent
- **Safety Filter (Port 8001)**: Content filtering and safety validation
- **Input Processor (Port 8002)**: Request preprocessing and validation
- **Response Generator (Port 8003)**: Response generation and formatting
- **Context Manager (Port 8004)**: Context storage and retrieval
- **Session Manager (Port 8005)**: Session state management
- **Audit Service (Port 8006)**: Audit logging and compliance
- **Metrics Collector (Port 8007)**: Metrics aggregation and reporting

### Enhanced Observability
- **OpenTelemetry Integration**: All services now use OpenTelemetry for distributed tracing
- **OTLP Protocol**: Unified observability endpoint at `http://localhost:4317`
- **Improved Metrics**: Better metrics collection and visualization
- **Distributed Tracing**: End-to-end request tracing across services

### Docker Compose Orchestration
- **Multi-container Setup**: All services managed via Docker Compose
- **Service Discovery**: Automatic service registration and discovery
- **Health Checks**: Built-in health monitoring for all services
- **Dependency Management**: Proper service startup ordering

## Breaking Changes

### Environment Variables
All services now require the `OTLP_ENDPOINT` environment variable:
```bash
OTLP_ENDPOINT=http://localhost:4317
```

### Port Changes
Ensure ports 8000-8007 are available in your environment. Previous services may have used different ports.

### Configuration Format
Service configuration has been standardized. Update your existing configuration files to match the new format.

## Configuration Changes

### New Environment Variables
Add the following to your `.env` file:
```bash
# OpenTelemetry Configuration
OTLP_ENDPOINT=http://localhost:4317

# Service Ports
BSL_AGENT_PORT=8000
SAFETY_FILTER_PORT=8001
INPUT_PROCESSOR_PORT=8002
RESPONSE_GENERATOR_PORT=8003
CONTEXT_MANAGER_PORT=8004
SESSION_MANAGER_PORT=8005
AUDIT_SERVICE_PORT=8006
METRICS_COLLECTOR_PORT=8007
```

### Docker Compose Changes
The new `docker-compose.yml` includes:
- All 8 new services
- OpenTelemetry Collector
- Service dependencies
- Health check configurations
- Network configuration

## Migration Steps

### Step 1: Backup Current Installation
```bash
# Backup your configuration
cp .env .env.backup
cp docker-compose.yml docker-compose.yml.backup
```

### Step 2: Update Environment Variables
Add the new environment variables to your `.env` file:
```bash
# OpenTelemetry endpoint
echo "OTLP_ENDPOINT=http://localhost:4317" >> .env

# Service ports
echo "BSL_AGENT_PORT=8000" >> .env
echo "SAFETY_FILTER_PORT=8001" >> .env
echo "INPUT_PROCESSOR_PORT=8002" >> .env
echo "RESPONSE_GENERATOR_PORT=8003" >> .env
echo "CONTEXT_MANAGER_PORT=8004" >> .env
echo "SESSION_MANAGER_PORT=8005" >> .env
echo "AUDIT_SERVICE_PORT=8006" >> .env
echo "METRICS_COLLECTOR_PORT=8007" >> .env
```

### Step 3: Update Docker Compose Configuration
Replace your existing `docker-compose.yml` with the new version that includes all 8 services.

### Step 4: Pull Latest Images
```bash
docker-compose pull
```

### Step 5: Stop Existing Services
```bash
docker-compose down
```

### Step 6: Start New Services
```bash
docker-compose up -d
```

### Step 7: Verify Deployment
```bash
# Check service status
docker-compose ps

# Check service logs
docker-compose logs -f

# Verify health endpoints
curl http://localhost:8000/health
curl http://localhost:8001/health
curl http://localhost:8002/health
curl http://localhost:8003/health
curl http://localhost:8004/health
curl http://localhost:8005/health
curl http://localhost:8006/health
curl http://localhost:8007/health
```

### Step 8: Verify Observability
```bash
# Check OpenTelemetry collector
curl http://localhost:4317/health

# View metrics (if using Prometheus)
curl http://localhost:9090/metrics
```

## Rollback Instructions

If you encounter issues and need to rollback to v0.3:

### Step 1: Stop All Services
```bash
docker-compose down
```

### Step 2: Restore Previous Configuration
```bash
cp .env.backup .env
cp docker-compose.yml.backup docker-compose.yml
```

### Step 3: Remove New Images (Optional)
```bash
docker rmi chimera-bsl-agent:v0.4.0
docker rmi chimera-safety-filter:v0.4.0
docker rmi chimera-input-processor:v0.4.0
docker rmi chimera-response-generator:v0.4.0
docker rmi chimera-context-manager:v0.4.0
docker rmi chimera-session-manager:v0.4.0
docker rmi chimera-audit-service:v0.4.0
docker rmi chimera-metrics-collector:v0.4.0
```

### Step 4: Restart Previous Version
```bash
docker-compose up -d
```

### Step 5: Verify Rollback
```bash
docker-compose ps
docker-compose logs -f
```

## Troubleshooting

### Port Conflicts
If you encounter port conflicts:
```bash
# Check what's using the ports
netstat -tulpn | grep -E '800[0-7]|4317'

# Update ports in .env file if needed
```

### Service Not Starting
```bash
# Check service logs
docker-compose logs <service-name>

# Check resource availability
docker stats
```

### Observability Issues
```bash
# Verify OpenTelemetry collector is running
docker-compose logs otel-collector

# Check OTLP endpoint connectivity
curl -v http://localhost:4317
```

### Database Migration
If you have existing data:
```bash
# Backup data volumes
docker run --rm -v chimera_data:/data -v $(pwd):/backup alpine tar czf /backup/data-backup.tar.gz -C /data .

# Restore if needed
docker run --rm -v chimera_data:/data -v $(pwd):/backup alpine tar xzf /backup/data-backup.tar.gz -C /data
```
