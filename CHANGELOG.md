# Changelog

All notable public changes to Project Chimera are documented here.

## Unreleased

- Added Phase 1 close-out review documents under `docs/closeout/` for
  submission readiness, case study framing, replication, evidence indexing, and
  claim status.
- Tightened close-out wording so the public repository supports the local
  adaptive AI demonstrator claim without asserting a completed public show,
  livestream, formal accessibility testing, BSL/avatar delivery, audience
  impact, complete evidence pack, or final payment readiness.

## v1.0.1-phase1 - 2026-05-06

- Published the cleaned public Phase 1 baseline.
- Kept the public claim narrow: local adaptive AI demonstrator, CLI, lightweight
  web console, public-safe docs, and validation helpers.
- Added and enforced public/private separation for grant records, receipts,
  invoices, participant data, generated evidence, tokens, and local `.env`
  files.
- Added maintained local documentation link checking.
- Fixed Dockerfile validation to match the current service baseline.
- Gated stale historical workflows to manual dispatch while keeping the active
  public checks on `main`.
- Applied GitHub repository metadata, branch protection, and stale branch
  cleanup.

## v1.0.0-phase1

- Marked the Phase 1 local adaptive AI foundation milestone.
- Documented Student / Laptop setup as the default reviewer route.
- Documented DGX Spark / GB10 and Kimi routes as advanced routes that require
  matching hardware and credentials.

## Earlier Experimental History

- Earlier March 2026 tags record exploratory service work and deployment ideas.
  Those tags remain available for project history, but the current public
  baseline is the Phase 1 local demonstrator described in `README.md`.
